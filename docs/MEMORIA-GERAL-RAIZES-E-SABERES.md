# Memoria geral - Plataforma Digital Raizes e Saberes

Este arquivo concentra o resumo do chat pesado "Criar plataforma educacional" para permitir continuar o projeto em chats menores, sem perder as decisoes ja tomadas.

## Projeto

- Nome: Plataforma Digital Raizes e Saberes.
- Objetivo atual: construir um showroom comercial do Ecossistema Educacional Raizes e Saberes, substituindo gradualmente prototipos por telas premium definitivas.
- Projeto local atual: `/Users/danielhenrique/Documents/plataforma digital raizes e saberes`.
- Chat antigo principal: `Criar plataforma educacional`.
- O trabalho tambem envolve GitHub e Vercel para publicacao.
- Ja havia mencao de FlutterFlow e Supabase no inicio do chat antigo, mas os detalhes completos dessa parte ainda precisam ser extraidos em uma segunda rodada se forem necessarios.

## Metodo oficial de trabalho

A Direcao do Projeto alterou oficialmente o fluxo:

1. O Design Studio cria telas premium finais em Desktop, Notebook, Tablet e Mobile.
2. O QG revisa e aprova cada tela.
3. O Codex implementa imediatamente a tela aprovada, sem aguardar as demais.
4. O Codex nao deve reinterpretar layouts nem criar novas propostas visuais quando houver tela aprovada.
5. Cada tela aprovada vira referencia visual absoluta para implementacao.

## Missao continua - Curadoria do Acervo Digital

A arquitetura da plataforma esta consolidada. A partir deste ponto, o Codex assume tambem a funcao de Curador do Acervo Digital.

Sempre que um novo material for aprovado pelo QG, o Codex deve:

1. Publicar o material na Biblioteca.
2. Integrar ao Book Viewer quando houver paginas reais disponiveis.
3. Atualizar automaticamente a colecao correspondente.
4. Atualizar o contador de materiais.
5. Destacar novidades publicadas.
6. Manter a organizacao do acervo e o Design System aprovado.

Nao e mais necessario solicitar ajustes estruturais individualmente para cada novo livro. O foco passa a ser a expansao continua do acervo, preservando exatamente o padrao ja aprovado para Biblioteca e Book Viewer.

## Missao continua - Universidade Viva

A Universidade Raizes e Saberes passa a funcionar como ambiente vivo, preparado para receber continuamente novos cursos reais.

Sempre que um novo curso for aprovado pelo QG, o Codex deve:

1. Publicar o curso na Universidade.
2. Atualizar Destaque do Curso da Semana quando aplicavel.
3. Exibir o curso entre os novos cursos publicados.
4. Atualizar automaticamente numero de cursos, horas de formacao e certificados emitidos.
5. Sinalizar trilhas em expansao com selo discreto "Em expansao".
6. Manter a estrutura preparada para videos reais, PDFs, avaliacoes e certificados.
7. Preservar integralmente o Design System e a experiencia visual aprovada.
